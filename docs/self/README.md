# Self Documentation

本目录用于记录项目开发过程中自定义添加的功能和改进。

## 文档列表

- **[features.md](./features.md)**: 功能开发记录
  - iQuant 实盘交易集成
  - 已修复的 Bug 和已实现的功能
  - 使用流程和技术架构说明

## 文档更新原则

1. **及时更新**: 每次实现新功能并验证通过后，立即更新相应文档
2. **详细记录**: 包括功能描述、技术实现、使用方法、已知问题等
3. **版本追踪**: 记录实现时间、相关提交、修改的文件
4. **代码对应**: 确保文档内容与代码实现保持一致

## 维护规范

- 使用 Markdown 格式
- 添加清晰的目录结构
- 使用代码块展示关键代码片段
- 添加流程图和表格辅助说明
- 标注功能状态（✅ 已完成、🚧 进行中、⚠️ 已知问题）
